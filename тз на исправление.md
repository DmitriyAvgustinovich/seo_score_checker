## Evidence rules

Evidence должна быть конкретной и основанной на DOM/pageData.

Примеры:

### Missing meta description

Evidence:
No meta description tag was found on this page.

Confidence:
High

### Missing title

Evidence:
The document title is empty or missing.

Confidence:
High

### Noindex

Evidence:
A robots meta directive contains noindex.

Confidence:
High

### Canonical to another URL

Evidence:
Canonical URL points to a different URL: <canonical URL>

Confidence:
High

### Missing H1

Evidence:
No H1 heading was found on this page.

Confidence:
High

### Multiple H1

Evidence:
Found <n> H1 headings on this page.

Confidence:
Medium

### Invalid JSON-LD

Evidence:
At least one JSON-LD script could not be parsed.

Confidence:
High

### Many images without alt

Evidence: <n> of <total> images are missing alt text.

Confidence:
Medium

### Open Graph missing

Evidence:
Open Graph tags were not detected.

Confidence:
Low

### Twitter tags missing

Evidence:
Twitter card tags were not detected.

Confidence:
Low

---

## Confidence rules

Use confidence labels:

High
Medium
Low

High confidence examples:

* noindex found
* title missing
* meta description missing
* canonical missing/invalid
* canonical points to another URL in a risky way
* missing H1
* invalid JSON-LD
* viewport missing

Medium confidence examples:

* title too short/long
* meta description too short/long
* multiple H1
* weak heading order
* many images without alt
* schema absent
* viewport issues

Low confidence examples:

* Open Graph missing
* Twitter tags missing
* lang missing
* charset missing
* unusual link counts
* social metadata only

Do not let Low confidence issues become High Traffic Risk.

Do not let social-only issues become High Traffic Risk.
